### 发布顺序

| 优先级 | 语言 | 原因 | 时间线 |
|--------|------|------|--------|

| P1 | 日文 | 日本AI社区活跃，AdSense RPM高 | 第2-3周 |

| P2 | 德文 | 欧洲高价值流量 | 第2-3月 |
| P3 | 法文 | 补充欧洲覆盖 | 第3-4月 |
| P3 | 阿拉伯文 | 中东市场，竞争极少 | 第4-6月 |

### 翻译策略

**核心原则：不要机翻，要本地化**

1. **技术术语**：保留英文（如 GitHub、API、AI Agent），本地用户都懂
2. **文化适配**：中文用"智能体"，英文用"AI Agent"，日文用"エージェント"
3. **hreflang**：每页都要有 `<link rel="alternate" hreflang="xx">`
4. **URL结构**：子目录式 `/en/`、`/ja/`、`/ko/`，不搞子域名（SEO权重分散）
5. **翻译工具**：
   - 初稿：DeepL（比Google翻译更自然）
   - 润色：本地母语者或AI二次优化
   - 校对：用 Grammarly（英文）/ LanguageTool（多语言）

### 检查清单

```
□ 每种语言有独立的 title、description、OG标签
□ hreflang 标签正确对应
□ canonical 指向对应语言版本
□ 站点地图包含所有语言版本URL
□ 语言切换菜单在固定位置（右上角）
□ 默认语言自动检测（浏览器语言）
□ 手动切换后记住偏好（localStorage）
```

## 六、GitHub Pages + Dynadot 自动部署

### 手动部署步骤（现在就能做）

```bash
# 1. 在 GitHub 创建仓库（用你的账号）
# 2. 把代码推上去
cd ~/clawdocs/projects/玩转ai进化网
git init
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git add .
git commit -m "init: 玩转AI进化网 MVP"
git push -u origin main

# 3. GitHub 设置 → Pages → 选择 main 分支
# 4. 等待几分钟，访问 https://your-username.github.io/your-repo/
```

### 自定义域名绑定（用 Dynadot）

**DNS 配置**：

```
# A记录（根域名）
@ → 185.199.108.153
@ → 185.199.109.153
@ → 185.199.110.153
@ → 185.199.111.153

# AAAA记录（IPv6，可选）
@ → 2606:50c0:8000::153
@ → 2606:50c0:8001::153
@ → 2606:50c0:8002::153
@ → 2606:50c0:8003::153

# CNAME记录（www子域名）
www → your-username.github.io
```

**GitHub 仓库配置**：

1. 在仓库根目录创建 `CNAME` 文件，内容是你的域名：
   ```
   aievoweb.com
   ```
2. 推送后，GitHub Pages 自动识别
3. 仓库 Settings → Pages → Custom domain → 输入你的域名
4. 勾选 "Enforce HTTPS"

### 自动部署脚本（GitHub Actions）

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
```

### 回滚方案

1. **Git 回滚**：`git revert HEAD` 或 `git reset --hard <commit>`
2. **GitHub Pages 回滚**：在仓库 Settings → Pages 中切换分支或版本
3. **DNS 回滚**：Dynadot 中删除 CNAME，恢复为默认域名

### 验证方式

```bash
# 验证域名解析
dig aievoweb.com +short

# 验证 HTTPS
curl -I https://aievoweb.com

# 验证 sitemap
curl https://aievoweb.com/sitemap.xml
```

## 八、广告上架时机

### 申请条件自检

在满足以下条件前**不要申请 AdSense**，否则会被拒：

| 条件 | 最低标准 | 推荐标准 | 当前状态 |
|------|---------|---------|---------|
| 网站上线时间 | ≥2周 | ≥1个月 | 刚上线 |
| 原创页面数量 | ≥15页 | ≥25页 | 7页 |
| 每页内容量 | ≥800字 | ≥1000字 | 部分内容不足 |
| 导航/政策页 | 完整 | 完整 | ✅ |
| 无侵权内容 | 100% | 100% | ✅ |
| 自然流量 | 有帮助 | 50-100/天 | 待积累 |
| 网站被索引 | 在Google能搜到 | 大部分页面被索引 | 待提交 |

### 建议申请时间

- **最早**：上线后 2 周，有 15+ 页内容，日均 50 访问
- **推荐**：上线后 1 个月，有 25+ 页内容，日均 100 访问
- **最安全**：上线后 2 个月，有 40+ 页内容，日均 200 访问

### 申请后如果被拒

1. 读拒绝邮件，定位具体问题
2. 修正后等待至少 1 个月再申请
3. 考虑先用替代广告网络（如 Ezoic、Mediavine）过渡
